Veille : La sécurité du cloud et l'intervention de l'IA dans ce domaine

2011 - 2013 : L'Audit et l'Identité

Concentration sur la gestion des accès. AWS IAM (2011) introduit la gestion fine des identités. AWS CloudTrail (2013) permet de tracer chaque appel d'API pour auditer les opérations en cloud.

Sources : Documentation AWS IAM ; Rapport Gartner sur l'IAM (2011) ; Annonce officielle AWS CloudTrail (2013) ; Security at Scale: Logging in AWS (Livre blanc).

2011 - 2013

2014 - 2019 : Le Zero Trust et l'IA Prédictive

Introduction du modèle Zero Trust par Google BeyondCorp (2014). Amazon Macie (2017) utilise le Machine Learning pour découvrir et protéger automatiquement les données sensibles (PII) dans le cloud.

Sources : BeyondCorp: A New Approach to Enterprise Security (Google Research) ; Forrester Wave: Zero Trust eXtended Ecosystem ; Lancement d'Amazon Macie (2017) ; TechCrunch: AWS launches Macie to secure data.

2014 - 2019

2020 - 2024 : L'IA Générative et la Posture

L'IA passe à l'assistance en langage naturel. Wiz (2020) utilise des graphes corrélés par IA pour identifier les chemins d'attaque. Microsoft Security Copilot (2024) avec GPT-4 permet aux analystes de poser des questions en français.

Sources : Wiz Case Study (2022) ; Forbes: How Wiz became the fastest-growing software company ; Microsoft Security Copilot General Availability (Avril 2024) ; Introduction to Copilot for Security (Cloud Direct).

2020 - 2024

2025 - 2026 : L'Ère de l'IA Agentique

L'IA devient un agent autonome capable d'agir seul. Amazon Bedrock AgentCore (2025) reconfigure les infrastructures en cas d'attaque. AI-Native SOC (2026) est géré à 90% par des IA capables de bloquer les menaces Zero Day.

Sources : AWS re:Invent 2025: Era of Agentic AI ; AWS Bedrock Agents Documentation ; CrowdStrike 2026 Global Threat Report ; The State of AI in the Cloud 2025 (Wiz Report).
